# OCRfanyi — 实时屏幕截图翻译工具

一个 Windows 桌面应用，框选屏幕任意区域后自动 OCR 识别文字并翻译成中文，翻译结果以透明浮层直接叠加在原文字位置。

## 效果演示

框选翻译区域 → 按热键 → 翻译结果直接覆盖在原文位置，鼠标可穿透，不干扰正常操作。

支持**自动刷新模式**：持续监控选中区域，文本变化时自动重新翻译（适合游戏字幕、视频、漫画等场景）。

## 功能

- **框选翻译**：鼠标拖拽选择屏幕任意区域，一键翻译
- **自动刷新**：持续监控选中区域，文本变化时自动重新翻译
- **双热键**：可自定义翻译热键和隐藏热键，单手操作
- **语言切换**：支持英语→中文 / 日语→中文，热切换无需重启
- **透明浮层**：翻译结果以透明窗口叠加在原文字位置，鼠标可穿透
- **暂停模式**：可随时暂停释放热键，恢复原按键功能
- **本地 OCR**：模型内置，无需联网，开箱即用

## 快速开始

### 方式一：下载即用（推荐）

1. 从 [Releases](https://github.com/chaonenglin/OCRfanyi/releases) 下载 `OCRfanyi.rar`
2. 解压到任意位置
3. 双击 `OCRfanyi.exe` 运行
4. 输入你的 [DeepSeek API Key](https://platform.deepseek.com/api_keys)（点"显示"确认后自动保存到环境变量）
5. 点击「框选翻译区域」选择屏幕区域
6. 按翻译热键或开启自动刷新即可

> **系统要求**：Windows 10+，NVIDIA 显卡（GTX 系列以上），CUDA 驱动

### 方式二：从源码运行

```bash
git clone https://github.com/chaonenglin/OCRfanyi.git
cd OCRfanyi
pip install torch --index-url https://download.pytorch.org/whl/cu126
pip install easyocr numpy mss Pillow aiohttp pywin32 pynput
set DEEPSEEK_API_KEY=你的API密钥
python app.py
```

## 快捷键

| 操作 | 默认按键 | 说明 |
|------|---------|------|
| 翻译 | `R` | 对选中区域执行 OCR + 翻译 |
| 隐藏 | `T` | 隐藏翻译浮层 |
| 自定义 | 点击界面中的「自定义」 | 按任意键替换默认热键 |
| 暂停 | 点击「暂停」按钮 | 释放热键，恢复按键原功能 |

## 技术栈

| 组件 | 技术 |
|------|------|
| OCR | EasyOCR (GPU, craft_mlt_25k + english_g2 / japanese_g2) |
| 翻译 | DeepSeek Chat API |
| 图像处理 | PIL/Pillow（灰度、增强对比度 1.8×、锐化、自动放大） |
| 屏幕捕获 | MSS |
| 浮层窗口 | Win32 API（ctypes）：`WS_EX_LAYERED` + `WS_EX_TRANSPARENT` + `WS_EX_TOPMOST` + `WS_EX_NOACTIVATE` |
| 热键 | `RegisterHotKey` + `MOD_NOREPEAT`，消息窗口线程 |
| UI | tkinter + PIL 合成背景 + Canvas 透明文字 |
| 打包 | PyInstaller `--onedir` |

## 配置说明

API Key 优先级：`DEEPSEEK_API_KEY` 环境变量 > 界面输入框

编辑 `config.py` 可调整：
- `LANGUAGES` — 语言列表（可添加更多语言）
- `CAPTURE_INTERVAL_MS` — 自动刷新间隔
- `FONT_SIZE_RATIO` — 翻译文字大小比例
- `OCR text_threshold / low_text` — 识别灵敏度

## 碎碎念

做这个工具最初是因为想看一些没有中文的生肉内容——切窗口、打字、等翻译太慢，截图丢进翻译软件又要来回切换。想要的是一个「框一下，翻译就浮在屏幕上」的东西，按一下就能用。

调试 EasyOCR 和 Win32 透明窗口的过程挺折磨的——PaddleOCR 在 Windows 上装不上，PyTorch CUDA DLL 各种依赖报错，tkinter 透明色穿透问题绕了好几圈。但最后能用自己写的工具翻译出一行日语的时候，确实挺开心的。

因为我不是程序员，所以代码里看不懂的地方可能不少。但管他呢，能用就行。自己用着爽最重要。
